HTML

An element is a type of content on a webpage. HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage. An HTML attribute defines additional information about an element. HTML attributes are always defined in the beginning element, never the closing element.

Inside the "head" we put the metadata

  • Declarations! aren't case sensitive but HTML are. (case sensitive)
  • title-defines the title for our webpage. useful for search engine optimization (SEO)
  • style-defines the CSS styles associated with a website
  • base-defines the base URL for a webpage
  • link-connects to an external resorce to the HTML document
  • meta-defines metadata such as character set, description, keywords, author, and veiwport
  • script-embeds client-side scripts in the HTML document
  • /head-head close tag (any tag is closed by using the /)
  • charset-character encoding
  • http-equiv-sets an http header for content
  • veiwport-default veiw for our webpage

The "body"

body-is a semantic element, it is important to use semantics so your HTML is easier to read by other deverlopers and the browser. It's also important for accessibility. Not all elements are semantic, but it's best to use semantic elements whenever possible.

Examples of non semantic elements:

  • div-to create a division on the page
  • span-to create and inline division

When working with visible HTML elements, the content is contained between an opening tag and a closing tag, These tags are most often represented by angle brackets with associated tag names contained inside. We put content inbetween these so the browser knows how to treat them. (like writing a paper)

  • h1-first header. h1,h2,h3... are the level of heading. (organizes your text so its easier to read.)
  • p-is a paragragh it formats the text to wrap on the webpage
  • ul,ol,and il- represent: ul unordered list, ol ordered list, and li list items. Can represent as a numbered list or bullet points.
  • img-information about images that are displayed on the webpage the "src" attribute is most important because it says the location of the image. The "alt" attribute contains a text string that describes the appearance and functionality of the image.
  • br-creates line break.
  • a-(stands for anchor) creates inks to the same webpage or other webpages. The "href" points to the "url" for the link
  • section- elements are semantic elements that define a section in a dociment that contains thematic content.
  • id- is a way to identify elements and select them.
  • rel- attribute specifies relationship between current doc and linked doc
  • href- is using relative path (being location of the linked doc from the location of the current doc, like going from your house to the store)
  • ./assets/style.css- current location/ file linked doc is located/linked doc

CSS

  • Inline CSS- style an element directly, they apply to a specific tag. useful for quick perminant changes, each style must be edited individually
  • internal style sheet- embed an entire style sheet into the HTML file, contains CSS rules that only apply to that page. you can configure with CSS and IDs, that identify elements, to style multiple things. a single change will apply to all tagged elements
  • external CSS style sheet- most common. This is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier. when using we link the file using the "link" element placed in the HTML's
  • declarations- (the rules) contains the CSS property we want to apply and the value of the property
  • selector- defines the element or attributes to which the rules, or declorations, will apply
  • margin- indicates the space we want around the outside of the element
  • padding- adds space around the inside of an element. (when padding is increased the element gets bigger)
  • star-declare all elements on the page will have the style properties listed
  • multiple elements can be grouped together using a comma if they you want them to have the same styling
  • DRY code-Do not repeate yourself is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained. DRY can greatly reduce the overall size and complexity of a codebase.
  • text-align-is a way of changing the alignment of text on a page
  • img- assigns property values to any image element in the markup
  • display-is assigned with the block value wich assigns each image to start on a new line
  • Class attribute- allow us to share a CSS rule to any element we choose by assigning a rule to a class attribute
  • width- width of the element
  • border-assigns a border with specified width
  • box-shadow-adds a shadow effect around element's frame. a box shadow is describes by x and y offsets relitive to the element, blur and spread radius, and color
  • assigning colors in CSS can be done by using semantic references such as gray or blue. we can also refer to a color by its hexadecimal code, or hex#
